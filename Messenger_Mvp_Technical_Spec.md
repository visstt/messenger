# Техническое задание: MVP мессенджера

## 1. Цель проекта

Разработать MVP веб/мобильного мессенджера с базовым функционалом обмена сообщениями между пользователями. На первом этапе приложение должно поддерживать личные чаты, текстовые сообщения, голосовые сообщения, отправку фото, просмотр истории переписки, статусы доставки/прочтения и базовую авторизацию.

Функции звонков, видеозвонков, каналов, сторис, ботов, платежей, сложной модерации и публичных групп на MVP-этапе не реализуются.

---

## 2. Основная идея продукта

Пользователь может зарегистрироваться, найти другого пользователя, начать с ним личный чат и обмениваться:

* текстовыми сообщениями;
* голосовыми сообщениями;
* фотографиями;
* несколькими фото в одном сообщении;
* ответами на сообщения;
* отредактированными или удалёнными сообщениями.

Приложение должно работать в реальном времени: новые сообщения должны появляться без перезагрузки страницы/экрана.

---

## 3. MVP-границы проекта

### 3.1. Что входит в MVP

В MVP необходимо реализовать:

1. Регистрацию и авторизацию пользователей.
2. Профиль пользователя.
3. Список чатов.
4. Личные чаты 1 на 1.
5. Отправку и получение текстовых сообщений.
6. Отправку и получение голосовых сообщений.
7. Отправку и получение фотографий.
8. Хранение истории переписки.
9. Статусы сообщений:

   * отправляется;
   * отправлено;
   * доставлено;
   * прочитано;
   * ошибка отправки.
10. Уведомление о новых сообщениях внутри приложения.
11. Индикатор непрочитанных сообщений.
12. Поиск пользователей.
13. Базовый поиск по своим чатам.
14. Удаление сообщений.
15. Редактирование текстовых сообщений.
16. Ответ на сообщение.
17. Просмотр фото в увеличенном виде.
18. Адаптивный интерфейс для desktop и mobile.

### 3.2. Что не входит в MVP

На первом этапе не реализовывать:

* голосовые звонки;
* видеозвонки;
* групповые чаты;
* каналы;
* сторис;
* реакции на сообщения;
* стикеры;
* GIF;
* ботов;
* секретные чаты с end-to-end encryption;
* сквозное шифрование;
* пересылку сообщений;
* закреплённые сообщения;
* папки чатов;
* архив чатов;
* публичные ссылки на профиль;
* сложную систему ролей;
* админ-панель;
* push-уведомления через FCM/APNs;
* Telegram-like Premium-функции.

## 4. Роли пользователей

### 4.1. Гость

Гость — пользователь, который не авторизован.

Может:

* открыть страницу входа;
* зарегистрироваться;
* восстановить пароль, если функция включена в реализацию;
* прочитать публичную страницу, если она есть.

Не может:

* просматривать чаты;
* искать пользователей;
* отправлять сообщения;
* просматривать чужие профили.

### 4.2. Авторизованный пользователь

Авторизованный пользователь может:

* просматривать список своих чатов;
* искать других пользователей;
* начинать личный чат;
* отправлять текст, голосовые сообщения и фото;
* редактировать свои текстовые сообщения;
* удалять свои сообщения;
* просматривать свой профиль;
* менять имя, username и аватар;
* выходить из аккаунта.

## 5. Основные экраны приложения

## 5.1. Экран регистрации

### Назначение

Создание нового аккаунта пользователя.

### Элементы интерфейса

На экране должны быть:

* поле имени;
* поле username;
* поле email или телефона;
* поле пароля;
* поле подтверждения пароля;
* кнопка «Зарегистрироваться»;
* ссылка «Уже есть аккаунт? Войти»;
* блок отображения ошибок.

### Поведение

Пользователь вводит данные и нажимает «Зарегистрироваться». Система проверяет:

* имя не пустое;
* username уникальный;
* email/телефон уникальный;
* пароль достаточной длины;
* пароль и подтверждение совпадают.

После успешной регистрации пользователь автоматически авторизуется и попадает на экран списка чатов.

### Ошибки

Отображать понятные сообщения:

* «Пользователь с таким username уже существует»;
* «Некорректный email»;
* «Пароль должен быть не менее 8 символов»;
* «Пароли не совпадают».

---

## 5.2. Экран входа

### Назначение

Авторизация существующего пользователя.

### Элементы интерфейса

* поле email/телефон/username;
* поле пароля;
* кнопка «Войти»;
* ссылка «Создать аккаунт»;
* опционально: ссылка «Забыли пароль?».

### Поведение

После ввода данных система проверяет их на сервере. При успешном входе пользователь попадает на экран списка чатов.

При ошибке показывать сообщение:

* «Неверный логин или пароль».

---

## 5.3. Экран списка чатов

### Назначение

Главный экран после авторизации. Показывает все личные переписки пользователя.

### Элементы интерфейса

* верхняя панель;
* аватар и имя текущего пользователя;
* кнопка перехода в профиль/настройки;
* строка поиска;
* список чатов;
* кнопка создания нового чата;
* индикатор подключения к серверу, если соединение потеряно.

### Карточка чата

Каждая карточка чата должна содержать:

* аватар собеседника;
* имя собеседника;
* username собеседника;
* последнее сообщение;
* время последнего сообщения;
* количество непрочитанных сообщений;
* иконку типа последнего сообщения:

  * текст;
  * голосовое;
  * фото;
* индикатор онлайн/офлайн собеседника, если реализуется.

### Поведение

При клике на чат открывается экран переписки.

Список чатов сортируется по дате последнего сообщения: самый активный чат находится сверху.

Если сообщений нет, показывать пустое состояние:

> У вас пока нет чатов. Найдите пользователя, чтобы начать общение.

---

## 5.4. Экран поиска пользователей

### Назначение

Позволяет найти другого пользователя и начать с ним чат.

### Элементы интерфейса

* поле поиска;
* список найденных пользователей;
* аватар пользователя;
* имя;
* username;
* кнопка «Написать».

### Поведение

Пользователь вводит имя или username. Система выполняет поиск по пользователям.

При нажатии на «Написать»:

* если чат уже существует — открыть существующий чат;
* если чата нет — создать новый чат и открыть его.

### Ограничения

Пользователь не должен видеть самого себя в результатах поиска.

---

## 5.5. Экран личного чата

### Назначение

Основной экран переписки между двумя пользователями.

### Элементы интерфейса

#### Верхняя панель

* кнопка «Назад» на мобильных устройствах;
* аватар собеседника;
* имя собеседника;
* username или статус;
* кнопка открытия информации о пользователе.

#### Область сообщений

* список сообщений;
* группировка сообщений по дате;
* разные стили для входящих и исходящих сообщений;
* автоскролл к последнему сообщению при открытии чата;
* подгрузка старых сообщений при прокрутке вверх.

#### Нижняя панель отправки

* кнопка прикрепления фото;
* поле ввода текста;
* кнопка записи голосового сообщения;
* кнопка отправки;
* превью выбранных фото;
* блок ответа на сообщение, если пользователь отвечает.

## 6. Сообщения

## 6.1. Типы сообщений

В MVP должны быть следующие типы сообщений:

1. `text` — текстовое сообщение.
2. `image` — одно или несколько фото.
3. `voice` — голосовое сообщение.
4. `mixed` — текст + фото, если решено поддержать в рамках MVP.

Минимально допустимо реализовать отдельно:

* текст;
* фото;
* голосовое.

---

## 6.2. Текстовые сообщения

### Возможности

Пользователь может:

* написать текст;
* отправить сообщение по кнопке или Enter;
* переносить строку через Shift + Enter;
* редактировать своё сообщение;
* удалить своё сообщение;
* ответить на сообщение.

### Ограничения

* Максимальная длина текстового сообщения: 4096 символов.
* Пустые сообщения не отправлять.
* Сообщение из одних пробелов не отправлять.

### Отображение

В сообщении показывать:

* текст;
* время отправки;
* статус доставки для исходящих сообщений;
* пометку «изменено», если сообщение редактировалось.

---

## 6.3. Фото-сообщения

### Возможности

Пользователь может:

* выбрать фото с устройства;
* отправить одно фото;
* отправить несколько фото;
* добавить текст к фото, если поддерживается;
* открыть фото в полноэкранном просмотре.

### Ограничения

* Поддерживаемые форматы: JPG, JPEG, PNG, WEBP.
* Максимальный размер одного файла: 10 МБ.
* Максимальное количество фото в одном сообщении: 10.

### Отображение

Фото должно отображаться в чате в виде превью.

При клике по фото открывается просмотрщик:

* затемнённый фон;
* изображение по центру;
* кнопка закрытия;
* переключение между фото, если их несколько.

### Обработка загрузки

Пока фото загружается, показывать:

* индикатор загрузки;
* временное превью;
* статус «отправляется».

При ошибке загрузки:

* показать статус «ошибка»;
* дать возможность повторить отправку.

---

## 6.4. Голосовые сообщения

### Возможности

Пользователь может:

* начать запись голосового сообщения;
* видеть длительность записи;
* остановить запись;
* отменить запись;
* отправить запись;
* прослушать голосовое сообщение в чате.

### UI записи

Во время записи показывать:

* таймер;
* кнопку отмены;
* кнопку отправки;
* визуальный индикатор записи;
* опционально: простую звуковую волну.

### Ограничения

* Минимальная длительность: 1 секунда.
* Максимальная длительность: 5 минут.
* Формат хранения: WebM/OGG/MP3/M4A — выбрать один основной формат в зависимости от платформы.
* Максимальный размер файла: 15 МБ.

### Отображение голосового сообщения

В чате показывать:

* кнопку Play/Pause;
* длительность;
* прогресс воспроизведения;
* время отправки;
* статус доставки.

### Ошибки

Обрабатывать ситуации:

* пользователь не дал доступ к микрофону;
* браузер/устройство не поддерживает запись;
* запись слишком короткая;
* ошибка загрузки файла на сервер.

## 7. Статусы сообщений

Каждое сообщение должно иметь статус.

### 7.1. Статусы

* `pending` — сообщение создаётся/загружается;
* `sent` — сообщение отправлено на сервер;
* `delivered` — сообщение доставлено получателю;
* `read` — сообщение прочитано получателем;
* `failed` — сообщение не отправлено.

### 7.2. Логика статусов

1. Пользователь нажал «Отправить».
2. Сообщение появляется в интерфейсе со статусом `pending`.
3. После ответа сервера статус меняется на `sent`.
4. Если получатель онлайн и получил событие через WebSocket — статус меняется на `delivered`.
5. Когда получатель открыл чат — статус меняется на `read`.

## 8. Реальное время

Для обмена сообщениями в реальном времени использовать WebSocket.

Допустимые технологии:

* Socket.IO;
* нативный WebSocket;
* SignalR, если backend на .NET;
* STOMP/WebSocket, если backend на Java/Spring.

### 8.1. События WebSocket

Минимальный набор событий:

#### Клиент → сервер

* `message:send` — отправка сообщения;
* `message:read` — пользователь прочитал сообщения;
* `typing:start` — пользователь начал печатать;
* `typing:stop` — пользователь перестал печатать;
* `voice:recording:start` — пользователь начал запись голосового;
* `voice:recording:stop` — пользователь остановил запись;
* `chat:open` — пользователь открыл чат;
* `chat:close` — пользователь закрыл чат.

#### Сервер → клиент

* `message:new` — новое сообщение;
* `message:updated` — сообщение отредактировано;
* `message:deleted` — сообщение удалено;
* `message:status_changed` — изменился статус сообщения;
* `typing:update` — собеседник печатает;
* `user:online` — пользователь онлайн;
* `user:offline` — пользователь офлайн;
* `chat:updated` — обновился список чатов.

## 9. Индикатор «печатает»

### Назначение

Показывает пользователю, что собеседник сейчас набирает сообщение.

### Поведение

Когда пользователь начинает вводить текст, клиент отправляет событие `typing:start`.

Если пользователь перестал вводить текст на 2–3 секунды, клиент отправляет `typing:stop`.

В чате собеседника отображается:

> Пользователь печатает...

## 11. Авторизация и безопасность

### 11.1. Регистрация

Пользователь создаёт аккаунт через email/телефон, username и пароль.

### 11.2. Вход

После успешного входа сервер возвращает access token и refresh token.

### 11.3. Хранение токенов

Рекомендуется:

* access token хранить в памяти приложения или безопасном хранилище;
* refresh token хранить в HttpOnly cookie, если используется веб;
* для mobile использовать secure storage.

### 11.4. Пароли

Пароли нельзя хранить в открытом виде.

Использовать хеширование:

* bcrypt;
* argon2;
* scrypt.

### 11.5. Доступ к чатам

Пользователь может получить доступ только к тем чатам, где он является участником.

Сервер обязан проверять права доступа на каждый запрос:

* получение чата;
* получение сообщений;
* отправка сообщения;
* удаление сообщения;
* редактирование сообщения;
* загрузка файлов.

---

## 12. Хранение файлов

Фото, аватары и голосовые сообщения не должны храниться напрямую в базе данных как бинарные данные.

Рекомендуемый подход:

1. Файл загружается на сервер.
2. Сервер сохраняет файл в объектное хранилище или локальное файловое хранилище.
3. В базе данных сохраняется ссылка на файл и метаданные.

### Варианты хранилища

Для MVP можно использовать:

* локальное файловое хранилище на сервере;
* S3-compatible storage;
* Cloudinary;
* Firebase Storage;
* Supabase Storage.

### Метаданные файла

Для каждого файла хранить:

* id;
* ownerId;
* originalName;
* mimeType;
* size;
* url/path;
* duration, если это голосовое;
* width/height, если это фото;
* createdAt.

## 14. API

## 14.1. Auth API

### POST `/api/auth/register`

Создание аккаунта.

#### Request

```json
{
  "name": "Ivan Ivanov",
  "username": "ivan",
  "email": "ivan@example.com",
  "password": "password123"
}
```

#### Response

```json
{
  "user": {
    "id": "uuid",
    "name": "Ivan Ivanov",
    "username": "ivan",
    "avatarUrl": null
  },
  "accessToken": "jwt-token"
}
```

---

### POST `/api/auth/login`

Вход в аккаунт.

#### Request

```json
{
  "login": "ivan",
  "password": "password123"
}
```

#### Response

```json
{
  "user": {
    "id": "uuid",
    "name": "Ivan Ivanov",
    "username": "ivan",
    "avatarUrl": null
  },
  "accessToken": "jwt-token"
}
```

---

### POST `/api/auth/logout`

Выход из аккаунта.

---

Получение текущего пользователя.

## 18. Нефункциональные требования

### 18.1. Производительность

* Голосовые сообщения не загружать полностью до нажатия Play, если возможно.

### 18.2. Адаптивность

Приложение должно корректно работать:

* на desktop;
* на планшетах;
* на мобильных устройствах.

На desktop можно использовать layout из двух колонок:

* слева список чатов;
* справа открытый чат.

На mobile:

* список чатов и чат открываются как отдельные экраны.

### 18.3. Надёжность

* При потере соединения показывать статус «Соединение потеряно».
* При восстановлении соединения автоматически обновлять список чатов и последние сообщения.
* Если сообщение не отправилось, пользователь должен иметь возможность повторить отправку.

### 18.4. Безопасность

* Все приватные API должны требовать авторизацию.
* Файлы должны быть доступны только участникам соответствующего чата или через защищённые URL.
* Проверять тип и размер загружаемых файлов.
* Защищаться от XSS: текст сообщений не должен вставляться как HTML.
* Ограничить частоту запросов на регистрацию, вход и отправку сообщений.

## 21. Критерии готовности MVP

MVP считается готовым, если:

1. Пользователь может зарегистрироваться.
2. Пользователь может войти в аккаунт.
3. Пользователь может найти другого пользователя.
4. Пользователь может создать личный чат.
5. Пользователь может отправить текстовое сообщение.
6. Второй пользователь получает сообщение без перезагрузки страницы.
7. Сообщения сохраняются в базе данных.
8. После обновления страницы история переписки не пропадает.
9. Пользователь может отправить фото.
10. Фото отображается в чате и открывается в увеличенном виде.
11. Пользователь может записать и отправить голосовое сообщение.
12. Голосовое сообщение можно воспроизвести.
13. В списке чатов отображается последнее сообщение.
14. В списке чатов отображается количество непрочитанных сообщений.
15. Сообщения имеют статусы отправки/прочтения.
16. Пользователь может редактировать своё текстовое сообщение.
17. Пользователь может удалить своё сообщение.
18. Пользователь не может читать чужие чаты.
19. Интерфейс работает на desktop и mobile.
20. Ошибки отображаются пользователю понятным образом.

Для username можно показывать подсказку:

```text
Username должен быть уникальным. Можно использовать латинские буквы, цифры и подчёркивание.
```

---

## Валидация

Ошибки должны появляться рядом с полями:

* имя пустое;
* username занят;
* email некорректный;
* пароль короткий;
* пароли не совпадают.

При успешной регистрации — переход в список чатов.

---

## Визуальные состояния

### Username проверяется

Показать маленький spinner справа внутри поля.

### Username свободен

Показать зелёную галочку.

### Username занят

Показать красную ошибку.

# 23.5. Главный экран: список чатов

## Назначение

Это основной экран приложения. Пользователь видит все свои переписки и быстро переходит в нужный чат.

---

## Desktop-вид

На desktop список чатов находится в левой колонке.

Пример:

```text
+----------------------------------+
| [Аватар] Messenger       [⚙]     |
|                                  |
| [🔍 Поиск]                       |
|                                  |
| Все чаты                         |
|                                  |
| [A] Иван Иванов          12:40    |
|     Привет, как дела?       2    |
|                                  |
| [M] Maria                11:18    |
|     🎤 Голосовое сообщение       |
|                                  |
| [D] Denis                Вчера    |
|     📷 Фото                      |
+----------------------------------+
```

---

## Верхняя панель списка чатов

Высота: 56–64px.

Содержит:

* аватар текущего пользователя;
* название приложения или имя пользователя;
* кнопку настроек/профиля;
* кнопку нового чата, если не вынесена отдельно.

Расположение:

```text
[Аватар] [Название/имя]                 [иконка]
```

Стиль:

* фон белый;
* нижняя граница 1px `#E5E7EB`;
* padding horizontal 12–16px.

---

## Аватар текущего пользователя

Размер: 36–40px.

Если фото нет:

* круглый фон с градиентом;
* первая буква имени;
* цвет текста белый.

Пример:

```text
И
```

---

## Строка поиска

Располагается под верхней панелью.

Контейнер:

* padding: 8–12px 12–16px;
* фон белый.

Поле поиска:

* высота 40–44px;
* border-radius: 999px или 12px;
* фон `#F3F4F6`;
* без сильной рамки;
* иконка лупы слева;
* placeholder: `Поиск`.

При фокусе:

* фон белый;
* outline акцентным цветом;
* можно показать кнопку очистки справа.

---

## Список чатов

Список занимает всё оставшееся пространство.

Должен быть вертикально прокручиваемым.

Каждый элемент списка — интерактивная карточка/строка.

---

## Карточка чата

Высота: 68–76px.

Структура:

```text
+------------------------------------------------+
| [Аватар]  Имя пользователя          Время      |
|           Последнее сообщение        [2]       |
+------------------------------------------------+
```

### Аватар

* размер 48–52px;
* круглый;
* слева;
* margin-right: 12px.

Если пользователь онлайн, можно показать маленькую зелёную точку:

* размер 10–12px;
* позиция снизу справа на аватаре;
* border 2px белый.

### Имя

* font-size: 15–16px;
* font-weight: 600;
* цвет `#111827`;
* одна строка;
* overflow ellipsis.

### Время

* font-size: 12px;
* цвет `#9CA3AF`;
* справа сверху.

Формат:

* сегодня: `14:32`;
* вчера: `Вчера`;
* старее: `05.05.26`.

### Последнее сообщение

* font-size: 13–14px;
* цвет `#6B7280`;
* одна строка;
* overflow ellipsis.

Для разных типов:

```text
Вы: Привет
📷 Фото
🎤 Голосовое сообщение
```

Если последнее сообщение удалено:

```text
Сообщение удалено
```

### Badge непрочитанных

Если есть непрочитанные сообщения:

* круг или pill;
* фон акцентный голубой;
* текст белый;
* min-width 20px;
* height 20px;
* font-size 12px;
* расположен справа снизу.

Пример:

```text
2
```

Если больше 99:

```text
99+
```

---

## Состояния карточки чата

### Default

Фон белый.

### Hover

Фон `#F3F4F6`.

### Active / выбранный чат

Фон светло-голубой `#E0F2FE`.

Имя может быть акцентным.

### Непрочитанный чат

* имя жирнее;
* последнее сообщение темнее;
* badge справа.

### Архивированный/отключённый

В MVP не нужен.

---

## Пустое состояние списка чатов

Если чатов нет:

```text
[иконка сообщения]
У вас пока нет чатов
Найдите пользователя, чтобы начать переписку
[Найти пользователя]
```

Визуально:

* центрировать внутри списка;
* иконка 48–64px;
* основной текст 16px semibold;
* вторичный текст 14px серый;
* кнопка голубая.

---

## Состояние загрузки

Показывать skeleton-строки:

* серые плашки вместо аватаров;
* серые линии вместо имени и последнего сообщения.

# 23.7. Экран открытого чата

## Назначение

Главный экран переписки между двумя пользователями.

---

## Desktop-структура

```text
+---------------------------------------------------------+
| [Аватар] Иван Иванов                                    |
| @ivan / был недавно                                     |
+---------------------------------------------------------+
|                                                         |
|                  область сообщений                      |
|                                                         |
|    [Входящее сообщение]                                 |
|                                  [Исходящее сообщение]  |
|                                                         |
+---------------------------------------------------------+
| [📎] [Введите сообщение...]              [🎤] [➤]       |
+---------------------------------------------------------+
```

---

## Верхняя панель чата

Высота: 56–64px.

Содержит:

* на mobile: кнопку назад;
* аватар собеседника;
* имя собеседника;
* статус;
* справа опционально кнопку меню.

Фон: белый.

Нижняя граница: `1px solid #E5E7EB`.

---

## Аватар собеседника

Размер:

* desktop: 40–44px;
* mobile: 36–40px.

Статус онлайн:

* маленькая зелёная точка;
* снизу справа.

---

## Имя и статус

Имя:

* 15–16px;
* font-weight: 600;
* цвет `#111827`.

Статус:

* 12–13px;
* цвет `#6B7280`;
* например:

  * `онлайн`;
  * `был недавно`;
  * `печатает...`.

Когда собеседник печатает, статус заменяется на:

```text
печатает...
```

Цвет можно сделать акцентным голубым.

---

## Область сообщений

Фон должен отличаться от боковой панели.

Варианты:

* светло-серый `#E7EBF0`;
* светло-голубой `#EFF6FF`;
* фон с очень слабым pattern, если хочется Telegram-like 느낌.

Для MVP лучше использовать однотонный фон.

Область должна:

* занимать всё доступное пространство;
* иметь вертикальный scroll;
* иметь padding 16–24px на desktop;
* иметь padding 10–12px на mobile;
* автоматически скроллиться вниз при новых сообщениях, если пользователь уже находится внизу.

---

## Дата-разделитель

Сообщения нужно группировать по датам.

Разделитель выглядит как маленький pill по центру:

```text
Сегодня
```

Стиль:

* background: rgba(255,255,255,0.75);
* border-radius: 999px;
* padding: 4px 10px;
* font-size: 12px;
* color: #6B7280;
* margin: 12–16px auto.

Варианты текста:

* `Сегодня`;
* `Вчера`;
* `5 мая 2026`.

---

## Пузырь сообщения

Сообщение должно быть в виде bubble.

### Входящие сообщения

Расположение: слева.

Фон: белый.

Цвет текста: `#111827`.

Скругление:

```css
border-radius: 18px 18px 18px 6px;
```

То есть нижний левый угол чуть менее скруглён.

### Исходящие сообщения

Расположение: справа.

Фон: мягкий зелёный или голубой.

Например:

```css
background: #D9FDD3;
```

Скругление:

```css
border-radius: 18px 18px 6px 18px;
```

---

## Размер bubble

* max-width desktop: 60–65% ширины области сообщений;
* max-width tablet: 70%;
* max-width mobile: 82–88%;
* min-width только по содержимому.

Padding:

* text message: 8px 10px;
* with media: 4–6px для фото + подпись отдельно.

---

## Текст сообщения

Стиль:

* font-size: 15–16px;
* line-height: 1.35–1.45;
* white-space: pre-wrap;
* word-break: break-word.

Ссылки, если автоопределяются:

* цвет акцентный;
* underline on hover.

---

## Время сообщения

Время отображается внутри bubble внизу справа.

Стиль:

* font-size: 11px;
* color: `#6B7280` для входящих;
* для исходящих можно чуть темнее/прозрачнее;
* margin-left: 8px;
* inline с текстом или отдельной строкой.

Пример:

```text
Привет, как дела?        12:40 ✓✓
```

---

## Статусы исходящих сообщений

Статусы отображать рядом со временем.

Варианты:

* pending: маленький spinner или часы;
* sent: одна галочка;
* delivered: две галочки;
* read: две галочки акцентного цвета;
* failed: красный восклицательный знак.

Пример:

```text
12:40 ✓✓
```

---

## Группировка сообщений одного автора

Если подряд идёт несколько сообщений от одного пользователя:

* между ними маленький отступ 2–4px;
* между группами разных пользователей отступ 8–12px;
* хвостик/особое скругление можно оставлять только у последнего сообщения группы.

Для MVP можно упростить: все сообщения имеют одинаковое скругление.

---

## Ответ на сообщение

Когда сообщение является ответом, внутри bubble сверху показывается маленький блок.

Вид:

```text
| Иван
| Привет, ты где?
Основной текст ответа
```

Стиль блока ответа:

* border-left: 3px solid акцент;
* padding-left: 8px;
* margin-bottom: 6px;
* background: rgba(255,255,255,0.35) или прозрачный;
* имя автора 12–13px semibold;
* текст превью 12–13px серый;
* максимум 1–2 строки;
* overflow ellipsis.

При клике на блок ответа можно проскроллить к исходному сообщению, но для MVP это опционально.

---

## Редактированное сообщение

Если сообщение редактировалось, рядом со временем добавить:

```text
изменено
```

Можно компактно:

```text
12:40 · изменено ✓✓
```

Стиль:

* font-size 11px;
* color secondary.

---

## Удалённое сообщение

Удалённое сообщение отображать как серый italic-текст:

```text
Сообщение удалено
```

Bubble можно оставить, но сделать:

* фон более светлый;
* текст серый;
* italic.

# 23.9. Голосовые сообщения UI

## Bubble голосового сообщения

Голосовое сообщение должно выглядеть как компактный аудио-плеер.

Пример:

```text
+------------------------------------+
| [▶]  ▂▃▄▅▃▂▂▃▄▆▅▃     0:12        |
|                         12:45 ✓✓   |
+------------------------------------+
```

---

## Элементы voice bubble

* кнопка Play/Pause;
* waveform или progress bar;
* длительность;
* время отправки;
* статус доставки.

### Кнопка Play/Pause

* круглая;
* размер 34–40px;
* фон акцентный для входящих или чуть темнее bubble для исходящих;
* иконка белая.

### Waveform

Для MVP можно сделать не настоящую волну, а декоративные вертикальные полоски разной высоты.

* высота области: 24–32px;
* цвет: серо-голубой;
* при проигрывании часть полосок можно окрашивать в акцентный цвет.

Если waveform сложно, использовать обычный progress bar:

```text
[▶] ━━━━━━━━━━━━━ 0:12
```

### Длительность

* font-size: 12–13px;
* цвет secondary.

Во время воспроизведения можно показывать текущее время:

```text
0:05 / 0:12
```

---

## Запись голосового сообщения

Когда пользователь нажимает кнопку микрофона, нижняя панель ввода меняется на режим записи.

Вид:

```text
[красная точка] 00:07       [Отмена]        [Отправить]
```

Или:

```text
[🗑]   ● 00:07  ▂▃▄▅▃▂▁▂▃      [➤]
```

---

## Состояния записи

### Recording

* красная точка мигает;
* таймер идёт;
* поле ввода скрыто;
* микрофон заменён на кнопку отправки или стоп.

### Cancel

При отмене:

* запись сбрасывается;
* панель возвращается в обычный режим.

### Permission denied

Если нет доступа к микрофону, показать toast или inline ошибку:

```text
Нет доступа к микрофону. Разрешите доступ в настройках браузера.
```

### Too short

Если запись меньше 1 секунды:

```text
Голосовое сообщение слишком короткое
```

# 23.10. Нижняя панель ввода сообщения

## Назначение

Пользователь вводит текст, прикрепляет фото и записывает голосовые сообщения.

---

## Общий вид

```text
+------------------------------------------------+
| [📎] [Введите сообщение...]        [🎤] [➤]    |
+------------------------------------------------+
```

---

## Контейнер

* фон: белый или прозрачный поверх фона чата;
* padding: 10–14px;
* border-top: 1px solid #E5E7EB;
* на mobile учитывать safe-area bottom.

```css
padding-bottom: calc(10px + env(safe-area-inset-bottom));
```

---

## Поле ввода

Textarea, а не input, чтобы поддерживать многострочный текст.

Высота:

* min-height: 40–44px;
* max-height: 120–160px;
* после max-height появляется scroll внутри textarea.

Стиль:

* background: `#F3F4F6`;
* border: none или лёгкая рамка;
* border-radius: 18–22px;
* padding: 10px 14px;
* font-size: 15px;
* resize: none.

Placeholder:

```text
Сообщение
```

---

## Кнопка прикрепления

Слева от поля.

* круглая область 40–44px;
* иконка скрепки;
* цвет secondary;
* hover background `#F3F4F6`.

При клике открывается меню вложений.

---

## Кнопка голосового

Если поле пустое — показывать кнопку микрофона.

Если есть текст или выбраны фото — показывать кнопку отправки.

Вариант поведения:

```text
пустое поле → [🎤]
есть текст → [➤]
```

---

## Кнопка отправки

* круглая;
* размер 40–44px;
* фон акцентный;
* иконка отправки белая;
* disabled, если нечего отправлять.

---

## Preview выбранных фото

Если пользователь выбрал фото до отправки, над input-панелью показать горизонтальный список превью.

Вид:

```text
+------------------------------------------------+
| [img x] [img x] [img x]                        |
| [📎] [Добавьте подпись...]             [➤]     |
+------------------------------------------------+
```

Превью:

* размер 56–72px;
* border-radius 10px;
* object-fit cover;
* кнопка `x` в углу для удаления.

Вид:

```text
+------------------------------------------------+
| [📎] [Введите сообщение...]       [➤]          |

```

Стиль:

* border-left акцентный;
* background `#F8FAFC`;
* padding 8–10px;
* имя автора semibold;
* текст превью серый;
* кнопка закрытия справа.

## Вид

# 23.16. Loading, empty и error states

## Loading

Использовать skeleton вместо глобального большого spinner там, где возможно.

### Список чатов loading

Показывать 6–8 skeleton-строк.

### Сообщения loading

Показывать несколько серых bubble-заглушек.

---

## Empty

Пустые состояния должны быть дружелюбными.

Пример:

```text
Здесь пока нет сообщений
Напишите первым
```

---

## Error

Ошибки должны быть понятными.

Пример:

```text
Не удалось загрузить сообщения
[Повторить]
```

Кнопка повторной попытки обязательна для критичных ошибок.

# 23.17. Mobile UI подробно

## Список чатов на mobile

Экран занимает всю ширину.

Верхняя панель:

```text
[Аватар] Чаты                    [поиск/новый]
```

Строка поиска может быть под заголовком.

Карточки чатов:

* height 68–74px;
* аватар 48px;
* меньше отступы;
* время и badge справа.

---

## Чат на mobile

Верхняя панель:

```text
[←] [Аватар] Иван       [⋯]
    был недавно
```

Высота: 56px.

Область сообщений:

* padding 8–10px;
* bubble max-width 86–90%;
* нижняя панель всегда закреплена снизу.

---

## Клавиатура

При открытии клавиатуры:

* поле ввода должно оставаться видимым;
* список сообщений должен уменьшаться по высоте;
* последнее сообщение не должно перекрываться клавиатурой.

---

## Mobile bottom input

* кнопки 40px;
* textarea min-height 40px;
* padding bottom с учётом safe area;
* preview фото должен быть горизонтально прокручиваемым.

---

# 23.18. Микроанимации

Анимации должны быть быстрыми и ненавязчивыми.

Рекомендуемые:

* hover для кнопок: 120–160ms;
* появление нового сообщения: лёгкий fade + translateY 4px;
* открытие модалки: fade overlay + scale 0.98 → 1;
* toast: slide/fade;
* spinner для загрузки;
* мигающая точка при записи голосового.

Не использовать тяжёлые анимации, которые мешают скорости мессенджера.

---

# 23.19. UX-правила поведения

## Отправка сообщения

При отправке сообщение должно сразу появляться в чате локально со статусом `pending`.

Пользователь не должен ждать ответа сервера, чтобы увидеть своё сообщение.

---

## Новые сообщения

Если пользователь находится внизу чата:

* новое сообщение автоматически появляется;
* чат скроллится вниз.

Если пользователь прокрутил историю вверх:

* не прыгать вниз автоматически;
* показать кнопку:

```text
Новые сообщения ↓
```

---

## Кнопка прокрутки вниз

Появляется, если пользователь не внизу.

Вид:

* круглая кнопка;
* справа снизу над input;
* иконка стрелки вниз;
* если есть новые сообщения — маленький badge.

---

## Повторная отправка

Если сообщение failed:

* показать красную иконку;
* при клике предложить `Повторить`;
* можно также показать `Удалить`.

---

## Контекстное меню сообщения

По правому клику на desktop или долгому нажатию на mobile показывать меню:

```text
Ответить
Редактировать
Удалить
```

Для чужих сообщений:

```text
Ответить
```

Для своих текстовых:

```text
Ответить
Редактировать
Удалить
```

Для своих фото/voice:

```text
Ответить
Удалить
```

## 24. Важные замечания для MVP

Для MVP не нужно пытаться полностью повторить Telegram. Главная задача — создать работающий фундамент мессенджера:

* аккаунты;
* личные чаты;
* сообщения;
* файлы;
* real-time;
* история;
* статусы.

После этого можно будет постепенно добавлять:

* групповые чаты;
* реакции;
* пересылку;
* поиск по сообщениям;
* push-уведомления;
* звонки;
* каналы;
* шифрование;
* админ-панель.
