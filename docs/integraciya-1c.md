# Интеграция с 1С

Идея подключения учётной системы для уведомлений в чат (поступление на склад, перемещения и т.п.).

---

## Рекомендуемая схема

1. **Служебный пользователь** в мессенджере (например «Склад-бот») — от его имени пишутся автоматические сообщения.
2. **Групповой чат** (или несколько чатов по складам), куда добавлены сотрудники и бот.
3. **Защищённый запрос** с сервера 1С на backend: «отправить текст в такой-то чат». Доступ по секретному ключу, не по паролю человека.
4. Backend сохраняет сообщение и **рассылает участникам** через WebSocket — как обычное новое сообщение.

### Как развести события по чатам


| Подход                 | Смысл                                      |
| ---------------------- | ------------------------------------------ |
| Один общий чат «Склад» | Все уведомления в одном месте              |
| Чат на каждый склад    | Разные чаты для Москвы, СПб и т.д.         |
| Разные типы документов | Приход, перемещение, заказ — разные тексты |


Маппинг «событие → чат → текст» — в конфиге или в таблице в базе.

### Надёжность и безопасность

- Ключ только на сервере 1С и на backend, не в браузере.
- По желанию — доступ только с IP сервера 1С.
- Лог уведомлений для разбора ошибок.
- При сбое сети — повтор или регламентное задание в 1С.

### Альтернативы прямому HTTP

- **Очередь сообщений** — 1С не ждёт ответа при проведении; отдельный сервис шлёт в мессенджер.
- **Периодический опрос** — проще по сети, но с задержкой.

Для складских сценариев обычно достаточно **отправки в момент проведения документа**.

---

## 1С — поверхностно

### Где цепляться

- Подписка на **проведение** нужных документов (приход, перемещение, реализация и т.д.).
- Либо обработчик после движений по складу.

### Что делает 1С

1. Фиксирует, что документ проведён (товар на складе / перемещён).
2. Собирает короткий текст для людей: документ, склад, позиции.
3. **Один раз** отправляет уведомление на сервер мессенджера.
4. При ошибке — журнал 1С; при необходимости — повтор регламентом.

### Что не нужно в 1С

- Встраивать интерфейс мессенджера в 1С.
- Хранить переписку в 1С — только триггер и текст.
- Логиниться в мессенджер под пользователем для каждого сообщения.

### Организационно

- Список документов и складов, которые дают уведомления.
- Создать чат(ы), добавить сотрудников.
- Передать ответственному URL сервера и секретный ключ (без хранения в открытом виде в конфигурации).

---

## Итог

Интеграция — **не мессенджер внутри 1С**, а **готовый текст в выбранный чат** после событий учёта.


| Сторона    | Роль                                            |
| ---------- | ----------------------------------------------- |
| Мессенджер | Бот, чат без E2EE, защищённый приём уведомлений |
| 1С         | Проведение документа → отправка уведомления     |
